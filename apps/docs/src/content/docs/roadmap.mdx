Mithril's docs describe only what's **verified against the source today**. This page tracks the
scoped work that isn't shipped yet, so you never hit a surprise `NOT_IMPLEMENTED`.

<Aside type="tip">
Everything under "Shipped" below is implemented and covered by the test suite. Everything under
"Next" and "Later" is designed but not yet built.
</Aside>

## Shipped today

- **Core** â€” the `MithrilEvent` protocol, total `reduce`/`replay`, `narrow()`, the streaming
  agent loop, and the `agentLoop` eject hatch.
- **Tools & agents** â€” typed tools via Standard Schema, three run methods (`run`/`stream`/`resume`).
- **Providers** â€” OpenAI, Anthropic, and Google adapters (real SSE), self-wiring handles,
  byok/proxy/ephemeral transports, plus **browser-local inference** via Transformers.js
  (`@mithril/providers/transformers` â€” on-device WebGPU/WASM, no keys).
- **Human-in-the-loop** â€” all three suspension tiers: Tier-1 approval (`needsApproval`), Tier-1b (a tool
  _returns_ `suspend(...)`), and Tier-2 (`await ctx.suspend()` mid-execute, resumed by replaying journaled
  effects). Resume with `resume()` (to a final `RunResult`) or `resumeStream()` (streaming), plus in-process
  `RunHandle.resolve()`. `seal`/`open` sealing (HMAC + AES-GCM) for persisted tokens.
- **Run control** â€” real `RunHandle.cancel()` (aborts at the next step boundary), `iterate()` step-level
  snapshots, and `asTool()` to call one agent as another's tool â€” with **first-class nested HITL**: a
  suspended sub-agent is resumed through the parent's own token.
- **Structured output** â€” validate â†’ retry, streamed `object.delta`.
- **Self-correction & reliability** â€” a **pluggable self-healing middleware stack** (`healing.defaults()`,
  toggled by the `healing` field): deterministic parse-repair + one visible schema coercion
  (`healing.argRepair` â†’ `tool.repair`), a bounded per-tool re-ask budget (`healing.retryBudget` â†’
  `tool.retry`, then a clear terminal error), identical-call **loop detection** (`healing.loopGuard` â†’
  `loop.detected`, steer-then-halt), structured-output re-ask (`healing.outputRetry` â†’ `object.invalid`),
  plus loop-level `outputSchema` enforcement, token/cost **budget guards**
  (`maxTokens`/`maxCostMicroUsd` â†’ `budget.exceeded`), a `ToolErrorClass` taxonomy on the wire,
  crash-hardening (provider/middleware throws â†’ typed `run.error`), and opt-in test-time compute
  (`selfConsistency`, `bestOfN`). See [Self-correction](/guides/self-correction/).
- **Composability** â€” **step-, model-, tool-, and finalize-altitude** middleware, plugins, event consumers.
- **Memory** â€” `Checkpointer` (in-memory + Bun SQLite) with a conformance kit.
- **Providers** â€” a Standard-Schema â†’ JSON-Schema converter (`toJsonSchema`/`withJsonSchema`) feeds real
  tool-parameter schemas to OpenAI/Anthropic/Google when the input schema self-describes or a converter is
  supplied.
- **MCP** â€” a client, an official Streamable-**HTTP transport** (`@mithril/mcp/http`), and **server
  exposure** (`@mithril/mcp/server`) to serve your own tools over MCP.
- **Devtools** â€” a headless inspector **and a framework-agnostic UI** (event stream, span tree, cost +
  context meters, time-travel scrubber, all-tier suspension card) shipped as a **Web Component**
  (`@mithril/devtools/element`), **mount functions** (`@mithril/devtools/dom`), and **React** components
  (`@mithril/devtools/ui`) over one rendering implementation â€” with **zero-touch attach**
  (`import "mithril/devtools/attach"` / `MITHRIL_DEVTOOLS=1`) and a `BroadcastChannel` live-tail panel.
- **Batteries** â€” OTel spans, `fs`/`kv` (in-memory + IndexedDB) adapters, the `vectors` store (in-memory +
  durable Bun SQLite), the `sandbox` code runner (`node:vm` + remote proxy), workflows, React bindings,
  `create-mithril`.
- **Studio (M1)** â€” a client-only studio app (`apps/studio`): design agents in structured panels **and** a
  fully two-way TypeScript code view (`@mithril/spec` â€” codegen + real-compiler parse with lossless
  verbatim regions), run them in-browser against live BYOK providers or on-device models via the shared
  worker runner (`@mithril/runner-web`, also powering the docs playground), with the devtools inspector,
  chat multi-turn, HITL approvals, and IndexedDB project persistence. See the
  [Studio guide](/guides/studio/).

## Later
