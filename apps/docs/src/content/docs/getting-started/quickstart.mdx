Let's build a weather assistant that calls a tool and then answers. You write three things — a
**tool**, an **agent**, and a **run** — and watch the typed event stream it produces. The example
below is complete and runs in the browser with **Open in playground**: no API key, no install.

## Build it

Here's the whole thing — a **tool**, an **agent**, and a **run**. This is real, shippable code:

```ts
import { agent, tool } from "mithril";
import { anthropic } from "mithril/anthropic";
import { z } from "zod"; // any Standard Schema validator

const weather = tool({
  name: "weather",
  description: "Current weather for a city.",
  inputSchema: z.object({ city: z.string() }),
  execute: async ({ city }) => ({ city, tempC: 21 }),
});

const assistant = agent({
  model: anthropic("claude-fable-5"), // self-wires the provider; key from ANTHROPIC_API_KEY
  instructions: "You are a concise weather assistant.",
  tools: [weather],
});

const result = await assistant.run("What's the weather in Istanbul?");
if (result.status === "completed") console.log(result.output);
```

- **The tool** is a name, a description the model reads, an `inputSchema` (any Standard Schema
  validator), and an `execute` function. Its input and output types are inferred for you.
- **The agent** takes a model, instructions, and its tools. The handle `anthropic("claude-fable-5")`
  self-wires the provider — no registry, just the `ANTHROPIC_API_KEY` env var.
- **The run** drives the loop. `assistant.run(input)` returns a `RunResult` you branch on by
  `.status`; `assistant.stream(input)` gives you tokens as they arrive (below).

### Run it here — no key

The exact same agent, but a **scripted model** stands in for the provider so it runs offline in your
browser. Click **Open in playground** — the tool call and answer stream into the inspector:

<Runnable title="quickstart.ts" highlight={["16-22"]} code={`import { agent, tool } from "mithril";
import { scriptedProvider, testModel } from "@mithril/core/testkit";
import { z } from "zod";

const model = testModel(scriptedProvider([
  [
    { type: "tool.call", callId: "c1", name: "weather", input: { city: "Istanbul" } },
    { type: "message.end", finishReason: "tool_calls", usage },
  ],
  [
    { type: "text.delta", delta: "It's 21°C and clear in Istanbul." },
    { type: "message.end", finishReason: "stop", usage },
  ],
]));

const weather = tool({
  name: "weather",
  description: "Current weather for a city.",
  inputSchema: z.object({ city: z.string() }),
  execute: async ({ city }) => ({ city, tempC: 21 }),
});

const assistant = agent({ model, instructions: "You are a concise weather assistant.", tools: [weather] });

await run(assistant, "What's the weather in Istanbul?");`} />

<Aside type="note" title="What changed for the playground?">
Only the model wiring. The `testModel(scriptedProvider(…))` block replaces `anthropic("claude-fable-5")`
so there's no network or key, and `run(agent, input)` / `usage` are playground globals that keep
snippets terse. In your own app you write the real version above — `assistant.run(...)`. See
[The playground](/getting-started/the-playground/) for the full list of injected globals.
</Aside>

## Watch the event stream

Prefer streaming? `stream()` returns a handle you iterate for plain text or typed events — same
agent, no `await` on the whole result:

```ts
const handle = assistant.stream("What's the weather in Istanbul?");
for await (const delta of handle.text) process.stdout.write(delta);
const { status } = await handle.result();
```
