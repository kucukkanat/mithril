The entire surface is three verbs: **`tool`**, **`agent`**, **`run`**. Everything past the 90%
path is _one more argument_ (highlighted) — never one more import. Run it:

<Runnable title="escalate.ts" highlight={["11-16"]} code={`import { agent, tool } from "mithril";
import { scriptedProvider, testModel, toolCallTurn, textTurn } from "@mithril/core/testkit";
import { z } from "zod";

const weather = tool({
  name: "weather",
  description: "Current weather for a city.",
  inputSchema: z.object({ city: z.string() }),
  execute: async ({ city }) => ({ city, tempC: 21 }),
});

// Escalate by argument, not import — tools, then output, then use: middleware/plugins.
const model = testModel(scriptedProvider([toolCallTurn("weather", { city: "Istanbul" }), textTurn("It's 21°C.")]));
const assistant = agent({ model, instructions: "Use weather.", tools: [weather] });

await run(assistant, "Weather in Istanbul?");`} />

## Bind dependencies once

Apps with dependencies bind them a single time with `createHarness` (highlighted), so no definition
ever restates a generic — `ctx.deps` is typed everywhere:

<Runnable title="harness.ts" highlight={[5]} code={`import { agent, createHarness } from "mithril";
import { scriptedProvider, testModel, toolCallTurn, textTurn } from "@mithril/core/testkit";
import { z } from "zod";

interface AppDeps { users: Map<string, { name: string }> }
const { agent: buildAgent, tool } = createHarness<AppDeps>();

const lookup = tool({
  name: "lookup",
  description: "Find a user by id.",
  inputSchema: z.object({ id: z.string() }),
  execute: async ({ id }, ctx) => ctx.deps.users.get(id) ?? { name: "unknown" }, // ctx.deps: AppDeps
});

const model = testModel(scriptedProvider([toolCallTurn("lookup", { id: "42" }), textTurn("That's Ada.")]));
const app = buildAgent({ model, instructions: "Look up the user.", tools: [lookup] });
await run(app, "Who is user 42?", { deps: { users: new Map([["42", { name: "Ada" }]]) } });`} />

## Three ways to run

`run`, `stream`, and `iterate` are genuinely different shapes, not aliases:

| Method | Returns | Use it when |
| --- | --- | --- |
| `agent.run(input)` | `Promise<RunResult>` | You want the final answer. |
| `agent.stream(input)` | `RunHandle` (async-iterable events + text) | You want to render tokens as they arrive. |
| `agent.resume(token, decision)` | `Promise<RunResult>` | You're resuming a suspended run. |

The 90% path uses `run`. Reach down a level only when it isn't enough — and if you outgrow the
whole loop, [eject it](/guides/ejecting-the-loop/): `agentLoop` is ~100 readable lines emitting a
byte-identical event stream.

The factory trio is `{ agent, tool, plugin }`; `agentLoop` is the eject hatch one layer down.
