An agent binds a **model**, **instructions**, and (optionally) **tools** into a loop you can `run`
or `stream`. Everything past the 90% path is one more argument — never one more import.

```ts
import { agent } from "mithril";
import { anthropic } from "mithril/anthropic";

const assistant = agent({
  model: anthropic("claude-fable-5"),
  instructions: "You are a concise assistant.",
  tools: [weather],
  maxSteps: 8, // hard cap on model↔tool round-trips (default 16)
});
```

## Model handles vs. registry strings

A provider handle self-wires — `anthropic("claude-fable-5")`, `openai("gpt-4o")`,
`google("gemini-2.0-flash")` — so no registry is needed. Prefer a bare `"provider/model"` string
(for config-driven routing)? Pass a `providers:` registry at run time and the string resolves over
it. A string with no registry throws `NO_PROVIDER` — an [error that tells you the
fix](/concepts/errors/).

## Instructions: string or function

`instructions` is a `string`, or a function of the `RunContext` — sync or async — for
per-run system prompts built from `ctx.deps`:

```ts
const support = agent({
  model,
  instructions: (ctx) => `You help ${ctx.deps.company} customers. Be terse.`,
  tools: [lookupOrder],
});
```

Because it takes `ctx`, the prompt is computed fresh each run and never captured in the serialized
state — only its output is.

## Run it, or stream it

`run`, `stream`, and `resume` are different shapes, not aliases:

| Method | Returns | Use it when |
| --- | --- | --- |
| `agent.run(input)` | `Promise<RunResult>` | You want the final answer. |
| `agent.stream(input)` | `RunHandle` | You want to render tokens as they arrive. |
| `agent.resume(token, decision)` | `Promise<RunResult>` | You're resuming a [suspended](/guides/human-in-the-loop/) run. |

`RunResult` is a discriminated union — branch on `.status` (`"completed"`, `"suspended"`,
`"unresumable"`, `"error"`, `"cancelled"`); never a bare try/catch. `Input` is a `string` or an
`InputMessage[]`. With no dependencies, the options object is fully optional — the API key comes
from `ANTHROPIC_API_KEY` / `OPENAI_API_KEY`.

```ts
const result = await assistant.run("What's the weather in Istanbul?");
if (result.status === "completed") console.log(result.output);
```

## Continuing a conversation

The `string` form is shorthand for a single `user` message. To send **chat history**, pass an
array of messages instead — `InputMessage` is `{ role: "user" | "assistant"; content: string }`:

```ts
const history = [
  { role: "user", content: "What's the capital of France?" },
  { role: "assistant", content: "Paris." },
  { role: "user", content: "And its population?" },
] as const;

const { output } = await assistant.run(history); // or assistant.stream(history)
```

Two rules the type enforces:

- **No `system` role.** The system prompt comes only from `instructions` — keep it there, not in
  the history.
- **`content` is text.** Each `{ role, content }` seeds one message; anything non-string is skipped.

You own the transcript: after each run, append the model's answer to your array and pass the grown
array to the next call. Mithril holds no hidden session between runs — that's the [state =
`replay(log)`](/concepts/state-is-replay/) principle. Durable *resume* of a single suspended run is
a separate, explicit [Checkpointer](/guides/memory-and-checkpointing/) concern.

Run the **Chat history** preset in the [playground](/playground/) to watch a transcript seeded from
prior turns, then answered.

<Aside type="note">
Threading the history yourself is deliberate — it keeps context construction visible and in your
control, with no framework guessing what to include or hidden session to reason about.
</Aside>

## Run one now

A two-tool round-trip, scripted with zero network — watch `run.start → tool.call → tool.result →
text.delta → run.finish` in the inspector:

<Runnable title="calculator.ts" code={`import { agent, tool } from "mithril";
import { scriptedProvider, testModel } from "@mithril/core/testkit";
import { z } from "zod";

const model = testModel(scriptedProvider([
  [
    { type: "tool.call", callId: "c1", name: "add", input: { a: 2, b: 40 } },
    { type: "message.end", finishReason: "tool_calls", usage },
  ],
  [
    { type: "text.delta", delta: "2 + 40 = 42." },
    { type: "message.end", finishReason: "stop", usage },
  ],
]));

const add = tool({
  name: "add",
  description: "Add two numbers.",
  inputSchema: z.object({ a: z.number(), b: z.number() }),
  execute: async ({ a, b }) => ({ sum: a + b }),
});

const calculator = agent({
  model,
  instructions: "Use add for arithmetic, then state the result.",
  tools: [add],
  maxSteps: 4,
});

const result = await run(calculator, "What is 2 + 40?");
if (result.status === "completed") console.log(result.output);`} />

<Aside type="tip" title="Outgrew the loop?">
`agent()` is the batteries-included loop. When you need full control, [eject
it](/guides/ejecting-the-loop/) — `agentLoop` is ~100 readable lines emitting the identical stream.
</Aside>
