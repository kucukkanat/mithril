Add an `output` schema and the loop stops handing you a string. It parses the model's JSON,
validates it against the schema, and types `result.output` for you:

```ts
import { agent } from "mithril";
import { anthropic } from "mithril/anthropic";
import { z } from "zod";

const extractor = agent({
  model: anthropic("claude-fable-5"),
  instructions: "Extract the weather as JSON.",
  output: z.object({ city: z.string(), tempC: z.number() }),
});

const result = await extractor.run("It's 21°C in Istanbul.");
if (result.status === "completed") console.log(result.output); // typed: { city: string; tempC: number }
```

The offline version below scripts the model's JSON across two `text.delta` chunks so you can run it
with no key (`output` schema highlighted) — click **Open in playground**:

<Runnable title="structured.ts" highlight={[11]} code={`import { agent } from "mithril";
import { scriptedProvider, testModel } from "@mithril/core/testkit";
import { z } from "zod";

// The scripted model returns its JSON across two text.delta chunks; the loop assembles + validates it.
const model = testModel(scriptedProvider([[
  { type: "text.delta", delta: '{"city":"Istanbul",' },
  { type: "text.delta", delta: ' "tempC":21}' },
  { type: "message.end", finishReason: "stop", usage },
]]));

const extractor = agent({
  model,
  instructions: "Extract the weather as JSON.",
  output: z.object({ city: z.string(), tempC: z.number() }),
});

const result = await run(extractor, "It's 21°C in Istanbul.");
if (result.status === "completed") console.log("typed output:", result.output); // { city, tempC }`} />

## Validate, then retry

If the model returns JSON that fails the schema, the loop emits `object.invalid` and re-asks the
model with the validation error — up to `outputRetries` times (highlighted; default 2). Here the
first scripted turn is missing `tempC`, so the loop retries and the second turn validates:

<Runnable title="retry.ts" highlight={[16]} code={`import { agent } from "mithril";
import { scriptedProvider, testModel } from "@mithril/core/testkit";
import { z } from "zod";

const model = testModel(scriptedProvider([
  [
    { type: "text.delta", delta: '{"city":"Istanbul"}' }, // invalid: no tempC
    { type: "message.end", finishReason: "stop", usage },
  ],
  [
    { type: "text.delta", delta: '{"city":"Istanbul","tempC":21}' }, // valid on retry
    { type: "message.end", finishReason: "stop", usage },
  ],
]));

const extractor = agent({
  model,
  instructions: "Extract the weather as JSON.",
  output: z.object({ city: z.string(), tempC: z.number() }),
  outputRetries: 3,
});

const result = await run(extractor, "It's 21°C in Istanbul.");
console.log(result.status === "completed" ? result.output : result.status);`} />

## It streams, too

Structured output is not all-or-nothing. As the JSON forms, the run emits `object.delta` with the
partial value; when it validates, a single `object.final` carries the typed result (both highlighted).
Fold them yourself, or in React let [`useObject`](/guides/react/) do it for you:

<Runnable title="streamed-object.ts" highlight={["19-20"]} code={`import { agent } from "mithril";
import { scriptedProvider, testModel } from "@mithril/core/testkit";
import { z } from "zod";

const model = testModel(scriptedProvider([[
  { type: "text.delta", delta: '{"city":"Istanbul",' },
  { type: "text.delta", delta: ' "tempC":21}' },
  { type: "message.end", finishReason: "stop", usage },
]]));

const extractor = agent({
  model,
  instructions: "Extract the weather as JSON.",
  output: z.object({ city: z.string(), tempC: z.number() }),
});

const handle = extractor.stream("It's 21°C in Istanbul.");
for await (const e of handle.events) {
  if (e.type === "object.delta") console.log("partial:", e.partial); // paints as it streams
  if (e.type === "object.final") console.log("final:", e.value); // set once, validated
}`} />

<Aside type="note" title="Output plus tools">
`output` composes with `tools` — the model can call tools to gather facts, then return the final
structured object. The last assistant turn is what's parsed against the schema. In React,
`useObject(agent.stream(input))` returns `{ partial, value }` — the same two events, wired for you.
</Aside>
