Mithril genuinely runs in the browser — the [playground](/playground/) on this site executes the
real loop in a Web Worker. Against a cloud provider, the only thing that changes client-side is **how
the request reaches the provider** — that is the `transport` (below). Or you can go all the way and run
**the model itself in the browser**, so there's no request, no server, and no key at all — see
[Local inference](#no-server-run-the-model-in-the-browser).

## Three transports, three trust models

The `transport` (highlighted) is the only thing that changes client-side. The runnable uses the
scripted model so it needs none; against a real provider you pass one of the three:

<Runnable title="transports.ts" highlight={["4-12"]} code={`import { agent } from "mithril";
import { scriptedProvider, testModel, textTurn } from "@mithril/core/testkit";

// The three trust models (used with a real provider handle, not the scripted model below):
const byok = { kind: "byok", apiKey: "sk-…" };                              // key in the browser — dev / personal
const proxy = { kind: "proxy", baseUrl: "https://api.example.com/llm" };    // your server holds the key
const ephemeral = { kind: "ephemeral", baseUrl: "https://api.example.com/llm", token: async () => "short-lived" };

const model = testModel(scriptedProvider([textTurn("This page is about running agents in the browser.")]));
const assistant = agent({ model, instructions: "Summarize pages." });

// With openai("gpt-4o") you'd pass e.g. { transport: proxy }; the scripted model ignores it.
console.log((await assistant.run("Summarize this page.")).status);`} />

Omit `transport` entirely and BYOK is read from `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` — handy on a
server, but there is no such env in a browser, so pass one explicitly.

<Aside type="caution" title="BYOK exposes the key">
A BYOK key in client code is visible to the user — never ship it in a public app. Use **proxy**
(your server holds the secret) or **ephemeral** (short-lived, server-minted tokens) for anything
untrusted. BYOK is for local dev and personal tools.
</Aside>

<Aside type="tip" title="Try BYOK in the playground">
The [playground](/playground/) can run these examples against a real model: in the **Run against** bar,
pick a live provider (OpenAI, Anthropic, Google, or Groq) and paste a key. It's pure BYOK — the key
stays in your browser and is sent straight to the provider, exactly as described here.
</Aside>

## CORS, handled

All four playground providers work directly from a browser with BYOK — you don't wire any of it:

- **OpenAI** serves CORS unconditionally, so the request just works.
- **Anthropic** auto-injects the `anthropic-dangerous-direct-browser-access: true` header, so the
  request isn't blocked. (The name of that header is a feature: it makes the key-exposure trade-off
  impossible to make by accident.)
- **Google** sends the key as a `?key=` query parameter to `generativelanguage.googleapis.com`, which
  serves permissive CORS for API-key calls.
- **Groq** speaks the OpenAI wire format (point `openaiProvider` at its `baseUrl`) and is CORS-friendly.

The provider does the right thing per host — Mistral/xAI are the ones that may still need a proxy.

## No server: run the model in the browser

Transports solve "how does the key reach a cloud model." The other option is to have **no cloud model**:
`@mithril/providers/transformers` runs an open model **on-device** with [Transformers.js](https://huggingface.co/docs/transformers.js)
(WebGPU, falling back to WASM/CPU). There's no transport, no key, and no network after the one-time weight
download — everything else about the agent is identical.

The API is identical to any provider — swap the model handle (highlighted) and the run behaves the
same. The runnable uses the scripted model so it's instant; to run the *real* on-device model, pick a
**Local · in-browser** model in the Run against bar (a one-time weight download):

<Runnable title="local.ts" highlight={[7]} code={`import { agent } from "mithril";
import { scriptedProvider, testModel, textTurn } from "@mithril/core/testkit";
// import { transformers, preload } from "@mithril/providers/transformers"; // on-device, no key
// await preload("onnx-community/Qwen3-0.6B-ONNX", { onProgress: (p) => setBar(p.progress) });

const model = testModel(scriptedProvider([textTurn("A concise summary.")])); // → transformers("onnx-community/Qwen3-0.6B-ONNX")
const local = agent({ model, instructions: "You are a concise assistant." });

const result = await local.run("Summarize this page."); // same event stream as any provider
console.log(result.status);`} />

`@huggingface/transformers` is an **optional peer dependency** — install it only in the app that uses local
inference. Weights are ~0.5–4.75 GB (feature-detect WebGPU; it falls back to WASM), and tool-calling on small
local models is **best-effort**. The full model catalog, worker recipe, and honest caveats are in the
[Local inference guide](/guides/local-inference/).

## Persist to OPFS

For durable client-side state — resumable tokens, cached transcripts — use the Origin Private File
System adapter. It is the same `FileSystem` interface as the Node and in-memory backends, so code
written against it ports unchanged:

The runnable uses the in-memory backend (highlighted); in a browser tab, swap it for
`opfsFileSystem()` — the same rooted interface, but durable and per-origin — to persist a suspended
token across reloads:

<Runnable title="opfs.ts" highlight={[1]} code={`import { memoryFileSystem } from "@mithril/fs"; // in a tab: import { opfsFileSystem } from "@mithril/fs/opfs"

const fs = memoryFileSystem(); // opfsFileSystem() is durable, per-origin, no permission prompt
await fs.mkdir("runs");
await fs.writeFile("runs/last.token", "a-suspended-token");
console.log("restored:", await fs.readText("runs/last.token"));`} />

Pair it with a [Checkpointer](/guides/memory-and-checkpointing/) to suspend a run in one tab and
resume it in another.

<Aside type="note" title="Rendering a live inspector">
Because every consumer needs only the event stream, you can render a live inspector in the browser
by iterating `handle.events`. The packaged UI ships at `@mithril/devtools/ui` — point
`<RunInspector source={handle} />` at your run, or `import "mithril/devtools/attach"` for zero-touch
capture. See [Devtools](/guides/devtools/).
</Aside>
