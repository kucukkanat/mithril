The React bindings are a thin store over the same `MithrilEvent` stream everything else consumes.
They are **headless** — no components, no styling, just state — so they slot into whatever UI you
already have.

## `useRun` — subscribe to a stream

Pass a `RunHandle` (from `agent.stream(input)`); `useRun` re-renders as events arrive:

```tsx
import { useMemo } from "react";
import { useRun } from "@mithril/react/hooks";

function Chat({ input }: { input: string }) {
  const run = useMemo(() => assistant.stream(input), [input]);
  const { text, status, costUsd } = useRun(run);
  return <div><pre>{text}</pre><small>{status} · ${costUsd.toFixed(4)}</small></div>;
}
```

`useRun` is a one-line wrapper over the DOM-free `createRunStore` (highlighted) — the runnable below
drives that store directly, giving the same `RunSnapshot` the hook returns
(`{ state, text, status, events, costUsd }`):

<Runnable title="run-store.ts" runnable={false} highlight={["8-12"]} code={`import { agent } from "mithril";
import { createRunStore } from "@mithril/react";
import { scriptedProvider, testModel, textTurn } from "@mithril/core/testkit";

const model = testModel(scriptedProvider([textTurn("It's 21°C in Istanbul.")]));
const assistant = agent({ model, instructions: "Be concise." });

const store = createRunStore(assistant.stream("Weather?").events);
store.subscribe(() => {
  const { text, status, costUsd } = store.getSnapshot();
  console.log(status, "·", JSON.stringify(text), "· $" + costUsd.toFixed(4));
});
await new Promise((r) => setTimeout(r, 50)); // let the scripted stream drain`} />

The `RunSnapshot` you get back is `{ state, text, status, events, costUsd }` — the folded
`RunState`, the concatenated text so far, a `status` (a `RunState` status, or `"streaming"` while
in flight), the raw `events`, and the running cost in USD.

## `useObject` — typed output as it forms

For a [structured-output](/guides/structured-output/) agent, `useObject` surfaces both the streaming
partial and the validated final value — the two events it folds are highlighted below (`useObject`
just wraps this fold with React state):

<Runnable title="use-object.ts" highlight={["13-14"]} code={`import { agent } from "mithril";
import { scriptedProvider, testModel } from "@mithril/core/testkit";
import { z } from "zod";

const model = testModel(scriptedProvider([[
  { type: "text.delta", delta: '{"city":"Istanbul",' },
  { type: "text.delta", delta: ' "tempC":21}' },
  { type: "message.end", finishReason: "stop", usage },
]]));
const extractor = agent({ model, instructions: "Extract JSON.", output: z.object({ city: z.string(), tempC: z.number() }) });

const handle = extractor.stream("It's 21°C in Istanbul.");
for await (const e of handle.events) {
  if (e.type === "object.delta") console.log("partial:", e.partial); // paints as JSON streams
  if (e.type === "object.final") console.log("value:", e.value); // set once, validated
}`} />

## `createRunStore` — the framework-agnostic core

Both hooks are one-line `useSyncExternalStore` wrappers over `createRunStore`, which is plain JS.
Use it directly outside React — Vue, Svelte, or a test — by subscribing to snapshots (highlighted).
There's also `createChatStore(agent)` for multi-turn chat, and `useChat` wraps it:

<Runnable title="agnostic.ts" runnable={false} highlight={["7-8"]} code={`import { agent } from "mithril";
import { createChatStore } from "@mithril/react";
import { scriptedProvider, testModel, textTurn } from "@mithril/core/testkit";

const model = testModel(scriptedProvider([textTurn("Hello! How can I help?")]));
const chat = createChatStore(agent({ model, instructions: "Be friendly." }));
chat.subscribe(() => console.log(chat.getSnapshot().status, "·", chat.getSnapshot().streaming));
chat.send("Hi there");
await new Promise((r) => setTimeout(r, 50)); // let the reply stream in
console.log("messages:", chat.getSnapshot().messages.length);`} />

<Aside type="note" title="Headless, one peer">
`@mithril/react` itself pulls in **no** React — the hooks live in the separate `@mithril/react/hooks`
entry so the core store type-checks and tests without a DOM. `react >= 18` is a peer dependency
(needed only for the hooks, which rely on `useSyncExternalStore`).
</Aside>
