There is exactly one source of truth ‚Äî the `MithrilEvent` stream. The three ways to consume a run
are progressively lower-level views of that same stream, so you reach for depth only when you need
it.

## Tier 1 ‚Äî the final result

Await it and branch on `.status`. No streaming machinery:

```ts
const result = await assistant.run("Summarize the incident.");
if (result.status === "completed") console.log(result.output);
```

## Tier 2 ‚Äî text as it arrives

`stream()` returns a `RunHandle`. Iterate `.text` for token deltas; await `.result()` for the final
`RunResult`:

```ts
const handle = assistant.stream("Summarize the incident.");
for await (const delta of handle.text) process.stdout.write(delta);
const result = await handle.result();
```

## Tier 3 ‚Äî every typed event

The handle is itself an async-iterable of events. `.state()` folds the buffered log into a
`RunState` at any moment:

```ts
const handle = assistant.stream("Summarize the incident.");
for await (const event of handle.events) {
  if (event.type === "tool.call") console.log("calling", event.name);
  if (event.type === "run.finish") console.log("done", event.usage);
}
```

## The event families

Every event is dot-namespaced and carries `EventMeta` (`v`, `runId`, `seq`, `ts`, `span`):

- **Lifecycle** ‚Äî `run.start`, `run.finish`, `run.error`, `run.cancel`, `step.start`, `step.finish`
- **Assistant body** ‚Äî `text.delta`, `reasoning.delta`, `tool.input.delta`, `tool.call`,
  `tool.progress`, `tool.result`, `tool.error`, `message.end`
- **Structured output** ‚Äî `object.delta`, `object.invalid`, `object.final`
- **Accounting** ‚Äî `usage`, `compaction`
- **Control flow** ‚Äî `handoff`, `tool.approval.requested`, `suspend`, `resume`
- **Escape hatch** ‚Äî `custom.${string}`

The union is [non-exhaustive by contract](/concepts/the-protocol/): a new event type is a minor
version, so switch on the types you care about and ignore the rest.

<Aside type="caution" title="Cancel via the signal, not the handle">
`handle.cancel()` is currently a **no-op**. To cancel a run, pass an `AbortSignal` in the run
options ‚Äî the timeout idiom is `signal: AbortSignal.timeout(30_000)`. Aborting emits `run.cancel`
and settles the result as `{ status: "cancelled" }`.
</Aside>
